Na pewno zdajesz sobie sprawę z tego czym jest konsola. Znacznie usprawnia proces testowania. Możemy w niej dodać komendy, które przyspieszą wykonanie określonych czynności, żeby móc przetestować różne scenariusze bez marnowania czasu na spełnienie poprzednich warunków, a przy złożonych projektach takich jest naprawdę dużo!

W moim przypadku, pierwszy taki projekt, który kwalifikował się pod dodanie konsoli była gra tworzona w ramach pracy inżynierskiej. Z racji, że praca była wykonywana w dwie osoby, gra musiała być odpowiednio duża. Z tyłu głowy miałem tą świadomość, że będzie dużo kluczy (korzystaliśmy z PlayerPrefs) i funkcjonalności.

Zacząłem szukać przykładu na to jak przygotować konsolę developera. Udało się trafić na pomysł od youtuber'a, Joey The Lantern. Konsolę Joey'a uzupełniłem o skupienie na polu do wpisywania komend po jej uruchomieniu, 'ściągania' ekranu konsoli do dołu w przypadku zwracania większej liczby informacji niż dostępna jest ilość ekranu (przeznaczonego na konsole), statyczne zmienne przechowujące popularne błędy, kolorowanie składni i predefiniowany design. Nie dodałem logów Unity, ale można to odnaleźć na kanale autora. Po projekcie zdecydowałem się udostępnić swoją wersję jako oddzielne repozytorium. Przykładowe komendy można znaleźć w szablonie (tutaj) lub w kodzie źródłowym gry Project Zero (tutaj).

Jak dodać komendę?

Stwórz klasę z nazwą CommandX gdze X to, krótki opis Twojej komendy (co by tradycji stało się zadość). Twoja wstępna wersja klasy mogłaby wyglądać następująco.

{% highlight cs linenos %} public class CommandX : ConsoleCommand { public sealed override string Name { get; protected set; } public sealed override string Command { get; protected set; } public sealed override string Description { get; protected set; } public sealed override string Help { get; protected set; } public sealed override string Example { get; protected set; } public CommandX() { Name = ""; // title of the command Command = ""; // text that will be recognized as command Description = ""; // give your command nice desc! Help = ""; // here you can put syntax for example Example = ""; AddCommandToConsole(); } public override void RunCommand(string[] data) { // put body of your command here // remember to properly secure // all cases, watch which pos // of array you are referring to } // creates command object public static CommandX CreateCommand() { return new CommandX(); } } {% endhighlight %}

W trakcie definiowania RunCommand nie zapomnij o tym, żeby zwrócić uwagę na typowe komunikaty konsoli przygotowane w klasie DevConsole. Jeżeli nie znajdziesz tej o której myślisz to przygotuj kolejną. Jak już komenda będzie gotowa to wystarczy wywołać statyczną metodę CreateCommand() nowej komendy w skrypcie DevConsole, w metodzie CreateCommands() (tak jak to pokazano niżej) i gotowe, można zabierać się do testowania!
