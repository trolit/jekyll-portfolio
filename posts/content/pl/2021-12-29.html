Pomyłki przy pracy z gitem zdarzają się dosyć często. Można np. zrobić commita na niewłaściwy mail. Sytuacja ta staje się problematyczna gdy zrobimy push do repozytorium. Taki commit da radę zmienić, ale jeżeli wcześniej się tego nie robiło to zdobycie informacji i poprawka może trochę zająć (chyba, że skorzysta się z tego rozwiązania t.j. zastosowania git rebase oraz commit z flagą amend) :P

Co należy zrobić?

  1. Przechodzimy do brancha, którego commity chcemy zmienić.
  2. Sprawdzamy o jaką wartość Z musimy się cofnąć, żeby się do nich dostać.
  3. Wykonujemy polecenie:
    {% highlight bash linenos %} git rebase -i HEAD~Z // (Z = 1, 2, ..., n + 1) {% endhighlight %}
  4. Dostajemy ostatnie Z commitów w edytorze.
  5. Przechodzimy na stronę repo -> brancha -> otwieramy listę commitów.
  6. Porównujemy identyfikatory commitów. Tym, które potrzebujemy zmodyfikować zmieniamy pick na edit (jeżeli korzystasz z edytora vi wciśnij A, żeby przejść do trybu edycji).
  7. Wychodzimy z trybu wprowadzania danych (w vi -> Escape).
  8. Zapisujemy plik i wychodzimy z edytora. Żeby to zrobić w vi, wpisz :wq! i naciśniij Enter
  9. Modyfikujemy --author w poniższym poleceniu i wykonujemy:
    {% highlight bash linenos %} git commit --amend --author="Fikcyjny Kowalski " {% endhighlight %}
    {% highlight bash linenos %} git rebase --continue {% endhighlight %}
    do moment aż nie ukaże się: Successfully rebased and updated... W między czasie mogą przeszkodzić konflikty oraz ekrany proszące o podanie komentarzy dla zmienianych commitów.
  10. Wykonujemy polecenie
    {% highlight bash linenos %} git push -f {% endhighlight %}
    Uwaga: Jeżeli zmiany dotyczą zamkniętego, niezmergowanego PRa, warto przed tym otworzyć ponownie PR <jeżeli jest zamknięty>. W przeciwnym wypadku będziemy musieli zrobić TEN zabieg, żeby móc otworzyć PR i żeby zmiany z lokalnego brancha od którego jest PR zostały uwzględnione w tym logu PRa.
  11. Gotowe :EZ:
